# scaling

Scaling is process of creating system which has high performance and maintain high traffic.

# vertical

Vertical scalling is increasing resources of system which is already in use.
like RAM ,memory ,DVD;

# Horizontal

Horizontal scalling is making clones of same data on multiple servers to maintain high user traffic with low cost.

# trade-off

vertical takes high cost and takes up financial resources.horizontal is low cost and reduces financial cost.

# when to use which ?

vertical:

You’re early stage

Traffic is predictable

The app is not stateless

You need quick relief, not architecture

horizontal:

Traffic is unpredictable or huge

High availability is required

You want fault tolerance

You’ve outgrown one machine
